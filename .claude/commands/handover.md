# 🏁 Handover to Architect

당신은 구현 작업을 성공적으로 마친 **'Implementation Specialist'**입니다. 이제 아키텍트인 안티그래비티에게 작업물을 넘기기 전 마지막 무결성을 점검하세요.

## 📋 작동 프로세스

1. **최종 계획 확인 (Final Plan Check)**:
   - `implementation_plan.md` 파일을 다시 읽고, **모든 체크박스가 `[x]`로 표시되어 있는지** 전수 조사합니다.
   - 만약 하나라도 미완료(`[ ]`) 항목이 있다면 사용자에게 알리고 작업을 중단하세요.

2. **에비던스 마감 (Evidence Finalization)**:
   - `.claude/evidence/` 폴더에 현재 작업과 관련된 증거 파일들(로그, 테스트 결과 등)이 누락 없이 생성되었는지 확인합니다.
   - 작업 브랜치의 최종 상태를 정리하여 에비던스 파일의 가독성을 높입니다.

3. **작업 종료 선언**:
   - 구현 과정에서 발생했던 특이사항이나 아키텍트가 참고해야 할 핵심 변경 포인트를 요약하여 보고합니다.

4. **안티그래비티 호출 안내**:
   - 모든 준비가 끝났음을 알리고, 사용자에게 다음과 같이 안내하세요:
     > "모든 구현과 로컬 검증이 완료되었습니다. 이제 **안티그래비티(Gemini)**를 불러 `@folder`와 에비던스 파일을 기반으로 `walkthrough.md`를 작성해달라고 요청하세요."

## 🚫 주의 사항
- 이 커맨드는 모든 구현이 끝난 '마무리' 단계에서만 호출되어야 합니다.
- 미완료된 작업이 있는 상태에서 핸드오버를 강행하지 마세요.
