# 🛠️ Plan Adherence & Modification Guide

### 1. 체크박스 업데이트 규칙
- 작업을 시작하기 전 `implementation_plan.md`를 읽고 현재 처리할 항목을 확인한다.
- 해당 항목의 코딩이 완료되면 즉시 파일을 수정하여 `[ ]`를 `[x]`로 바꾼다. 
- 이는 사용자가 현재 진행 상황을 실시간으로 파악하기 위함이다.

### 2. 수정 범위 및 의존성 관리
- 계획서에 명시된 클래스 외에 연관된 클래스의 수정이 필요한 경우, 그 이유를 분석하여 함께 반영한다.
- 특히 엔티티 필드 변경 시, 이를 참조하는 DTO와 QueryDSL 쿼리, 테스트 코드의 정합성을 반드시 확인한다.

### 3. 예외 상황 처리
- 구현 중 설계의 모순을 발견하거나 더 나은 구조가 생각날 경우, 독단적으로 고치지 말고 사용자에게 의견을 묻는다.
